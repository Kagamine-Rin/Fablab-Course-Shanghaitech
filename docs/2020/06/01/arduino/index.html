Playing with Arduino

Published on 2020 Jun 01 23:57:24
Last Updated on 2020 Jun 20 11:28:47

Playing with the Arduino and LCD 1602 screen

Background

I always love the old stuff (such as those computers that use 6502 CPU) so I liked the video produced by the Youtuber: The 8-bit guy.

As the old chips are no longer be produced now, he had to replace some old stuff with Arduino. He also made a video about LCD1602. So I got quite interested.

Preparation

First we have to figure out the address of this chip. This chip uses a driver chip modeled PCF8574AT (Some got PCF8574T). I tried the I2C scanner but sadly it did not have any output….

However after asking Google I managed to find the address: 0x3F. And this was a success!

(Strangely, according to Zeyu Zheng, the I2C protocol is a double-sided protocol, which means the I2C scanner should find the address if it’s properly connected - and it’s true. But why it’s not my case… Is it because of not well compatible with this Arduino 101 made by Intel?)

Also, I knew the chip is most likely to be produced by TOSHIBA and it should support displaying some Japanese characters. So I wrote a program to display ASCII code from 0 to 255, and their corresponding characters in the font library of this 1602 screen. See in github: 3/scanner.ino.

I was right - the katakana(Japanese: 片仮名) are arranged in Japanese alphabetical (Japanese: 五十音図) order.

Now I can show “Hello world” in 2 languages! Code: 3/helloworld.ino

Light meter

Since for now I only have the light sensor, I want to visualize how much light it receives by the LCD screen. Since it’s possible to define up to 8 custom characters, I will use 5 of them. They’ll be available at ASCII 0-7, and repeated at 8-15.

For convenience I will put a character maker here: link

And this:

Let’s do some maths: The output of light sensor range from 0 to 1023. The LCD has 16 columns and each cell has 5 columns - That’s mapping 1024 to 80. Also, the darker it gets, the high value - So I have to invert the result:
result = 1023 - result

Then that’s easy! This machine do the following thing: When the button is pushed, the amount of light that are detected by the sensor is shown on the screen, like a bar.

The code is in github: 3/light.ino

Perfect!
