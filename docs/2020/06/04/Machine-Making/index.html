Machine Making

Published on 2020 Jun 04 20:46:01
Last Updated on 2020 Jun 22 16:06:30

Using self-made machines to print a figure (although not that success)

Introduction

It is very interesting to make a custom machine that can draw or write something. I dreamed to have a robot for writing homework when I was very young - And it has become reality!

The framework of the machine is made during the lecture. However we have to add something to support a functionality to somehow lift the pen above paper.

Software Setup

First of all Inkscape must be installed (together with a plugin to generate Gcode files).

I have an image of Hatsune Miku who is my favorite “paper girlfriend”. I’ll just let the “printer” draw her.

Of course, I also made a simple image to test the “printer”.

Inkscape can be easily downloaded from the official website. Theoretically by installing a plugin, the gcode file can be easily made. However I have to made many attempts and finally I realized the version of the plugin was much too old. I switched to a new version and it finally works!

As there are many duplicated lines, I had to delete many of them. The final version looked like this:

I took a look at the generated Gcode file. It seems that there are really some problems with the plugin - It always duplicate the original trace and I had to remove the extra trace manually. But there are more problems:

  • Some gcode instructions are not recognized by the board;
  • The generated code uses M280 P0 S30 (pen down) for pen down and M280 P0 S50 (pen up) for pen up, but in our case we have to move the Z axis;
  • The “wait” instruction is actually in seconds - it should be in milliseconds! I once thought it’s stuck.

So I had to use a script to change it into the movement of Z-axis so it can work, remove the unrecognized instructions and fix the bug:

The complete code can be found at 4/change.py.

Trail

After the new grbl-servo firmware is uploaded, the original CNC shield also worked. I had to tune the parameters to $100=80.0, $101=80.0 as in class. But it went fine and the setup was also successful.

I let it “print” the test image and it looked like this:

I accidentally moved the paper so there was a displacement. Overall this is fine - But because of the pen holder is not very steady, the line is a little shaking like the above image shows.

So, finally I decided not to let it draw a miku - that will ruin my favorite face!!!!
